**シナリオ4：バックアップの自動化と世代管理（副業需要 ★★★★★）**
現場のトラブル: 「過去に操作ミスでデータを消してしまったことがある。現在は手動でバックアップを取っているが、取り忘れが発生するうえに、古いバックアップが削除されず放置され、ストレージ費用を無駄に圧迫している。」
ハンズオン手法（コスト数十円）:
テスト用のEC2インスタンス（またはEBSボリュームのみ）を作成し、タグに Backup = Daily を付ける。
AWS Backupのコンソールから新しい「バックアッププラン」を作成する。
バックアップルールで「毎日特定の時間にバックアップを取得し、7日経過したら自動削除する（ライフサイクル）」設定を行う。
「リソースの割り当て」で、タグ条件を Backup = Daily に指定する。
スケジュールを待たずに「オンデマンドバックアップ」を手動実行し、ジョブが成功して「復旧ポイント（バックアップデータ）」が作成されることを確認する。
確認後、作成された復旧ポイント、バックアッププラン、ならびにテスト用のEC2（またはEBS）を削除する。

### このシナリオのおすすめ理由（背景）
AWS Backup を使ったバックアップの自動化・一元管理は、**「オンプレミスからAWSに移行してきたばかりの中小企業」を中心に副業案件（環境構築・運用改善）で非常に多く依頼されるタスク**です。監視やコスト最適化とセットで提案しやすいため、習得しておくと単価アップやリピート受注に直結しやすい領域になります。
