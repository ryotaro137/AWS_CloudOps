# AWS CloudOps (SysOps) 実技ハンズオン優先順位表

CloudOps（SysOps Administrator）の試験範囲および実務でも最も重要視されるのは**「トラブルシューティング（障害対応・原因切り分け）」「可用性の確保」「ネットワークとセキュリティの適切な設定」**です。そのため、既存環境のトラブル対応や、モダンな環境（コンテナ・サーバーレス）への移行タスクが上位になります。

---

## 🥇 優先度：高（必須レベル・超重要）
CloudOpsのコアスキルである「トラブルシューティング」「ネットワーク切り分け」「ログ調査」が直接問われるシナリオです。

### 1位：Infrastructure trouble-shooting on AWS Vol.1
- **概要**: 掲示板サービスプロジェクトの問題解決。AWS にて運用する事が決定していたが、チームはAWSに不慣れ。構成の構築とデプロイを行なったところいくつかの問題が発生している状況。EC2、VPCを中心として構成されている。
- **選定理由**: EC2とVPCを中心としたトラブルシューティングは、CloudOpsの実務・試験において最も基礎かつ最重要なスキルです。「VPCルートテーブル」「セキュリティグループ」「ネットワークACL」「IAMロール」のどこに問題があるかを見つけ出し、修正する能力が直球で試されるため、優先度1位です。

### 2位：Infrastructure trouble-shooting on AWS Vol.2
- **概要**: コンテナベースのREST APIサーバのPoC構築。ローカルでは動作したが、AWS上のFargate/ECS環境に持ってきたところ動作しない状態。
- **選定理由**: Vol.1のコンテナ（ECS/Fargate）版です。EC2環境のトラブルシューティングができるようになったら、次は「タスクロール/タスク実行ロールの違い」「VPCエンドポイントの設定漏れ」「セキュリティグループの設定ミスでイメージがPullできない」といった、コンテナ特有のトラブルシューティング能力を確かめるのに最適です。

---

## 🥈 優先度：中（実践的な移行とモダン化）
ただの初期構築ではなく、既存のオンプレミスやレガシーな構成から「クラウドのメリットを活かした設計・運用」へのシフトを問われるシナリオです。

### 3位：Contact Page Migration Based on Serverless (Basic)
- **概要**: 軽量なお問い合わせページを自社サービスと同じEC2を間借りして運用していたが、EC2からECSに移行することになり、お問い合わせページも移行する必要がある。
- **選定理由**: 「EC2の間借りからサーバーレス/コンテナへのリファクタリング」という、非常に実務的なシナリオです。CloudOpsでは運用負荷を下げるためのモダナイゼーション（マネージドサービスの活用）が重要視されるため、この移行タスクは運用のスキル証明に結びつきます。

### 4位：EC Site Migration Based on Amazon EC2 (Basic)
- **概要**: AWS以外の環境で運用されてきたECサイトをAWSに載せ替える。
- **選定理由**: 「様々な要求に応えつつ実装」というプロンプトから、単なるEC2構築ではなく「Auto Scalingの設定」「ALBによる監視と負荷分散」「RDSのMulti-AZ」といった、CloudOpsに不可欠な高可用性・耐障害性の知識が試される良質なシナリオです。

---

## 🥉 優先度：低（設計・初期構築メイン）
CloudOpsのスキル（運用、監視、トラブル対応）よりも、Solutions Architect寄りの「初期構築」や「アーキテクチャ設計」の毛色が強いものです。

### 5位：Event Site Launch Based on Amazon ECS (Basic)
- **概要**: 利用者が自由にイベント登録出来るサービスを新規リリース。コンテナを使って実装。
- **選定理由**: 新規リリースに伴うECS環境の構築です。重要ではありますが、「運用トラブルの解決」よりも「ゼロからのデプロイ」能力が試されるため、CloudOpsスキルの確認としては優先度を下げています。

### 6位：WordPress Site Migration Based on Amazon EC2
- **概要**: レンタルサーバーにて運用中のWordPressサイトをAWSに移行する。
- **選定理由**: 4位のECサイト移行とコンセプトが似ていますが、特定のCMS（WordPress）に寄っているため、より汎用的なインフラ運用要件が試されるECサイト移行の方を優先すべきです。

### 7位：Hybrid Cloud Based on AWS Direct Connect（※設計のみ）
- **概要**: 物理サーバーのリース契約更新タイミングで、オンプレミスとAWSのハイブリッドクラウド構成を検討する。
- **選定理由**: 「アーキテクチャ設計試験のみ」であるため、ハンズオンでのネットワーク構築エラーの解決や、ルーティングのデバッグといった体験ができません。CloudOpsは「実際に手を動かして直せるか」が重視されるため、実技スキルを確かめる優先度としては最後尾になります。
