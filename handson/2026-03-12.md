# 2026-03-12 勉強記録：ハンズオンシナリオ実装（AWS CDKによるIaC基礎）

本日は未経験者向けの「AWS Cloud Development Kit (CDK)」の基礎的なハンズオンシナリオを設計しました。運用（CloudOps）において、インフラのコード化（IaC）は必須スキルです。さらに昨日の手動構築で体験した「ECSでのデプロイ時間の長さや手間の多さ」を、インフラ自動化によってどう解決できるかを体験できる構成になっています。

---

## 難易度：初級（★★☆☆☆）CDK First Step

### シナリオ10：Hello, CDK! - 平日30分×2日で学ぶ「インフラのコード化と自動デプロイ」

**【コンセプト】**
AWSコンソール画面からの手動操作から卒業し、プログラミング言語（今回は未経験でも扱いやすいTypeScriptを想定）を使ってAWSリソースを構築・削除する体験をします。コストがかからず、かつ実務でよく使われる「サーバーレス（API Gateway + Lambda）」の構成をCDKで構築し、IaCの強力さを手軽に体感する入門シナリオです。

---

#### ⬛️ Day 1（前半30分）：CDKの準備と初回デプロイ（空のスタック）
- **概要**: 開発環境でCDKプロジェクトを初期化し、AWS環境にデプロイの準備（Bootstrap）を行います。
- **目的**: IaCツールの基本的な使い方（initやbootstrap）を学び、AWSアカウントと手元のターミナルを紐付けます。
- **Day 1 の解決手順（ハンズオン）**:
  1. **【準備】**: 
     - Node.js がインストールされていることを確認。
     - ターミナルで `npm install -g aws-cdk` を実行し CDK をインストール。
  2. **【Step 1: プロジェクトの初期化】**:
     - 空のディレクトリを作成し、移動（例：`mkdir hello-cdk && cd hello-cdk`）。
     - `cdk init app --language typescript` を実行してスケルトンコードを生成します。
  3. **【Step 2: Bootstrapと初回デプロイ】**:
     - CDKがデプロイに必要なS3バケットなどのアセット用環境を作成する `cdk bootstrap` を実行。
     - そのまま `cdk deploy` を実行し、AWSの「CloudFormation」コンソール画面で空のスタックが作成されたことを確認します。
  4. **【Day 1 クリーンアップ】**:
     - 今回は空のスタックなのでコストは発生しませんが、終わったら `cdk destroy` を実行してスタックを削除する練習をします。

- **Day 1 ハンズオン費用目安**: **$0.00 (無料)**
  - クラウドリソースをほぼ作成しないため無料です。Bootstrap用のS3バケットも微小なデータを置くだけなので課金枠には入りません。

---

#### ⬛️ Day 2（後半30分）：APIとLambdaの一瞬デプロイ
- **概要**: Day 1で作った空のプロジェクトに数行のコードを追加し、「アクセスすると "Hello, CDK!" と返すAPI」を即座にデプロイします。
- **目的**: ECSなどのコンテナデプロイにおける「Pull時間や起動の長さ」に比べて、Lambda＋API Gatewayのサーバーレス構成がどれほど速く、かつCDKなら構築もコンソールを触らず一瞬で終わるかを体験します。
- **Day 2 の解決手順（ハンズオン）**:
  1. **【準備】**: 
     - プロジェクト内にLambda関数のコードを作成します（例：`lib/hello.js` に `exports.handler = async function() { return { statusCode: 200, body: "Hello, CDK!" }; };` を記述）。
  2. **【Step 3: CDKコードの記述】**:
     - `lib/hello-cdk-stack.ts` にAWSリソースを定義するコードを追記します。
     - Lambda関数の定義：`new lambda.Function(...)`
     - API Gatewayの定義：`new apigateway.LambdaRestApi(...)`
       （※通常なら面倒なIAMロールや権限設定も、CDKの「コンストラクト」が自動生成してくれます）
  3. **【Step 4: 差分確認とデプロイ】**:
     - `cdk diff` を実行し、ターミナル上で「どのAWSリソースが追加されるか」を事前に確認します（実務で非常に重要な安全確認）。
     - `cdk deploy` を実行。数分待つとターミナルにAPIのURL（Endpoint URL）が出力されるので、ブラウザやcurlでアクセスして "Hello, CDK!" が返ってくるか確認します。
  4. **【Day 2 最終確認と全クリーンアップ】**: （最重要）
     - 動作確認が終わったら、**必ず `cdk destroy` を実行**し、LambdaとAPI Gateway等のリソースを綺麗に一括削除します。手動設定での消し忘れや、依存関係エラーの心配がありません。

- **Day 2 ハンズオン費用目安**: **$0.00 (無料枠内)**
  - LambdaとAPI Gatewayはどちらも強力な無料枠があるため、個人のハンズオンレベルの数回アクセスでは一切コストはかかりません。

### 💡 選定理由（未経験者向けIaC入門の価値）
- **完全無料・リスクゼロ**: 消し忘れによる高額請求のリスクが全くないサーバーレスアーキテクチャを採用しています。
- **IaCの恩恵を最速で実感**: コンソールでポチポチとLambdaを作り、API Gatewayを作り、IAMを紐付ける…という面倒な作業とヒューマンエラーのリスクが、わずか数行の記述と `cdk deploy` 一発で完了する体験ができます。
- **副業需要への直結**: CDKやTerraformなどのIaCスキルは、CloudOpsやクラウド構築の副業案件において**必須かつ最も需要の高い（★★★★★）技術領域**です。この小さな成功体験による精神的ハードルの低下が、将来の高単価案件獲得の大きな第一歩となります。
