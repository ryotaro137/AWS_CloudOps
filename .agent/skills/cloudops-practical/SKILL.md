# CloudOps Practical Skill

CloudOps試験の学習内容を、副業で即使える実践知識と紐付けて教えるスキル。

## 回答フォーマット

AWSサービスや用語について質問された際、以下の3つの観点を必ず含めて回答する。

### 1. 試験での出題ポイント
- そのサービスが試験でどう問われるか
- よくある選択肢の引っ掛けパターン
- 類似サービスとの違い（比較問題対策）

### 2. 実務でのユースケース
- 現場でどのような構成で使われるか（他サービスとの連携パターン）
- 設計時の注意点やベストプラクティス
- トラブルシューティングのコツ

### 3. 副業案件での需要度
- そのスキルが副業市場でどの程度求められているか
- 具体的な案件例と単価感
- 案件獲得に繋がるアピールポイント

## 副業需要の優先度マップ

回答時に副業観点のアドバイスをする際、以下の優先度を参考にする。

| 優先度 | 分野 | 理由 |
|:---|:---|:---|
| ★★★★★ | 監視・アラート設計 | ほぼ全ての案件で必要。導入が比較的容易で成果が見えやすい |
| ★★★★★ | IaC (CloudFormation/Terraform) | 需要が最も高い。手動構築からの移行案件が大量にある |
| ★★★★ | コスト最適化 | 経営層に刺さりやすく、リピート率が高い |
| ★★★★ | セキュリティ運用 | コンプライアンス要件で必須。単価も高い |
| ★★★★ | CI/CD パイプライン | モダン開発チームからの需要が増加中 |
| ★★★ | ログ分析 | 障害調査時に重宝。ただし単体案件は少ない |

## 学習日記への追記フォーマット

回答内容を学習日記（`til/YYYY-MM-DD.md`）に追記する際は以下の形式を使用する。

```markdown
## [サービス名/トピック名]
- **概要**: [1〜2行での説明]
- **試験ポイント**: [試験で問われやすい要点]
- **実務での使い方**: [現場での構成パターンや設計のコツ]
- **副業での需要**: [需要度★と、よくある案件例]
```
