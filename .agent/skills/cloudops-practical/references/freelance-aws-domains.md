## 監視・アラート設計
**副業需要: ★★★★★（最高）**

### よくある案件例
- 「CloudWatchアラームを設計して、Slack通知まで繋ぎたい」
- 「本番環境の異常を自動検知する仕組みが欲しい」
- 「ダッシュボードを作って経営層に見せたい」

### 必要なサービス知識
- CloudWatch Metrics / Alarms / Dashboards（クロスリージョン含む）
- CloudWatch Logs Insights（クエリ構文）
- EventBridge（イベントパターン・フィルタリング）
- SNS（通知チャネル管理）
- CloudWatch Synthetics / RUM（外形監視）

### 副業での単価感
- 監視設計＋構築: 30〜80万円/案件
- ダッシュボード構築のみ: 10〜30万円/案件
- 継続的な運用監視: 月5〜15万円

### 実務でのポイント
- アラーム閾値は「ベースライン（通常時の値）」を計測してから設定する
- アラームが多すぎると「アラーム疲れ」が起きるため、重要度で分類してSNSトピックを分ける
- ダッシュボードは「経営層向け（ビジネスKPI）」と「エンジニア向け（技術指標）」を分けて設計する

## IaC (Infrastructure as Code)
**副業需要: ★★★★★（最高）**

### よくある案件例
- 「手動で構築した環境をCloudFormation/Terraformでコード化したい」
- 「新環境を素早く複製できるようにしたい」
- 「StackSetsで全アカウントに同一構成を適用したい」

### 必要なサービス知識
- CloudFormation（テンプレート構文、ドリフト検出、StackSets）
- AWS CDK / Terraform（CloudFormation以外の選択肢）
- AWS Config（設定の継続的な監視）
- SSM Parameter Store / Secrets Manager（パラメータ管理）

### 副業での単価感
- 既存環境のIaC化: 50〜150万円/案件
- テンプレート設計・レビュー: 20〜50万円/案件

### 実務でのポイント
- 既存環境のインポート機能やFormerを使ってテンプレートの雛形を作り、手作業を減らす
- ネストスタックはメリット・デメリットがあるため、チーム規模に合わせて判断する
- ドリフト検出を定期的に実行し、手動変更を見つけ次第テンプレートに反映する

## コスト最適化
**副業需要: ★★★★（高い）**

### よくある案件例
- 「AWS費用が急に上がったので原因分析と削減提案が欲しい」
- 「リザーブドインスタンスやSavings Plansの購入判断をしたい」
- 「Compute Optimizerの提案に基づいてリサイジングしたい」

### 必要なサービス知識
- Cost Explorer / Budgets / Cost Anomaly Detection
- Compute Optimizer（EC2, EBS, Lambda の最適化提案）
- Trusted Advisor（コスト最適化チェック）
- S3 Storage Lens / Intelligent-Tiering

### 副業での単価感
- コスト診断レポート: 10〜30万円/案件
- 最適化実施（RI購入支援含む）: 30〜80万円/案件
- 月次コストモニタリング: 月5〜10万円

### 実務でのポイント
- Compute Optimizer で CW Agent を入れていないとメモリ情報が取れないので注意
- コスト削減提案は「年間削減額」で表現すると経営層への説得力が増す
- タグ運用（コスト配分タグ）の設計が、コスト可視化の基盤になる

## セキュリティ運用
**副業需要: ★★★★（高い）**

### よくある案件例
- 「Security Hub / GuardDuty を導入してセキュリティを強化したい」
- 「AWS Configルールで全アカウントのセキュリティ基準を統一したい」
- 「インシデント対応自動化（Config + SSM Automation）を構築してほしい」

### 必要なサービス知識
- AWS Config（ルール、是正措置、コンフォーマンスパック）
- SSM Automation（自動是正ランブック）
- Security Hub / GuardDuty / Inspector
- CloudTrail（監査ログ）
- IAM（最小権限の原則、アクセスアナライザー）

### 副業での単価感
- セキュリティ基盤構築: 50〜120万円/案件
- Config + 自動是正の設計・実装: 30〜60万円/案件

### 実務でのポイント
- AWS Config + SSM Automation の自動是正は試験にも実務にも頻出の最重要パターン
- Security Hubのスコアを「改善前 → 改善後」で比較レポートとして出すと、成果が可視化される
- マルチアカウント環境ではOrganizations + 委任管理者の理解が必要

## CI/CD パイプライン
**副業需要: ★★★★（高い）**

### よくある案件例
- 「CodePipeline + CodeDeploy で自動デプロイ環境を構築してほしい」
- 「ECSのブルー/グリーンデプロイを実現したい」
- 「GitHub ActionsからOIDCでAWSにデプロイしたい」

### 必要なサービス知識
- CodePipeline / CodeBuild / CodeDeploy
- ECS デプロイ戦略（ローリング、ブルー/グリーン）
- OIDC（GitHub Actions / GitLab CI 連携）
- CloudFormation（パイプラインのIaC化）

### 副業での単価感
- CI/CDパイプライン構築: 30〜80万円/案件

### 実務でのポイント
- ブルー/グリーンデプロイはロールバックが容易で本番に強い
- OIDCを使うことで長期クレデンシャルをCI環境に保存しない構成が主流
- テスト環境 → ステージング → 本番の多段パイプラインは「承認ステップ」を挟む

## ログ分析・トラブルシュート
**副業需要: ★★★（中程度）**

### よくある案件例
- 「アプリのエラー原因を特定するためにログ基盤を整備したい」
- 「CloudWatch Logs Insightsでログを検索・分析したい」

### 必要なサービス知識
- CloudWatch Logs / Logs Insights（クエリ構文）
- X-Ray（分散トレーシング）
- CloudTrail（API操作の追跡）
- VPC Flow Logs（ネットワーク分析）

### 副業での単価感
- ログ基盤設計・構築: 20〜50万円/案件
- トラブルシューティング支援: 時給5,000〜10,000円

### 実務でのポイント
- ログは「構造化（JSON）」で出力すると Logs Insights で分析しやすい
- メトリクスフィルターとアラームを組み合わせてエラー検知を自動化する
- コスト削減のため、ログのライフサイクル（保持期間 + S3へのエクスポート）を設計する
